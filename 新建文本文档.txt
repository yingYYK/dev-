1. 分支：克隆一份 master 

2. git branch  ---->  查看分支

   git branch 分支名（有规则的名字） ---->  创建一个分支 

  	 一般情况下 名字为 dev ( 开发分支名 )  development

   	 但是 在项目开发的时候 会有很多 开发这个项目的人都会创建一个分支 
  
   	 那么 都取同一个 dev 是不可取的，所以命名是有规范的；  
   
   git branch -d 分支名  ----->  删除分支

3. git checkout 分支名  ----->  切换到分支

         但是 切换到分支之后， 工作区没有变化，
      
         但是 工作区 已经是 切换到的分支的工作区了

         由于工具的原因 是看不到 多个分支同时存在的， 只能操作到当前分支

         分支开发没有提交版本之前，即没有存进本地仓库之前，不能进行分支的切换

4. 把dev 分支 合并到 master 分支上

   git merge branch  ----> 把 分支名为 branch 的分支 合并到 当前分支
